# 项目看板 Skill

## 功能说明
从飞书多维表格读取项目数据，生成可视化HTML看板，并推送到GitHub Pages。

## 触发命令
```
/项目看板           # 更新看板并推送
/项目看板 --本地     # 仅更新本地HTML，不推送
```

## 执行步骤

### 步骤1：读取飞书数据
使用 `mcp__lark-mcp__bitable_v1_appTableRecord_search` 读取所有任务数据：
- 任务名称
- 所属项目（AI年度会员/爆款产品/ToB业务/求职课/素人矩阵/基础建设/人员招聘）
- 负责人
- 开始时间、结束时间
- 状态（未开始/进行中/已完成/有风险）
- 优先级（P0/P1/P2/P3）
- 是否里程碑
- 依赖任务
- 备注

### 步骤2：生成HTML看板
按以下结构生成HTML：

```
1. 页头
   - 标题：2026年度项目管理看板
   - 年度目标：5000万
   - 更新时间

2. 收入目标卡片
   - AI年度会员：2000-2500万
   - 爆款产品：800-1500万
   - ToB业务：300-500万
   - 求职课：300-400万
   - 素人矩阵：100-200万

3. 1-2月紧急任务（P0/P1任务列表）

4. 关键任务依赖链

5. 核心团队成员

6. 各项目甘特图（按所属项目分组）
   - 一、AI年度会员
   - 二、爆款产品-AI工具
   - 三、ToB业务
   - 四、求职课
   - 五、素人矩阵
   - 六、基础建设
   - 七、人员招聘

7. 关键里程碑（is_milestone=true的任务）

8. 页脚
```

### 步骤3：保存HTML
写入到 `reports/2026年度项目甘特图.html`

### 步骤4：推送GitHub（除非 --本地）
```bash
cd "E:\阿宝姐公司管理" && git add . && git commit -m "更新项目管理看板" && git push
```

## 甘特条样式
- `not-started`：灰色，未开始
- `in-progress`：橙色渐变+闪烁，进行中
- `completed`：绿色渐变，已完成
- `milestone`：粉紫色渐变，里程碑
