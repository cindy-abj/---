# 流量分析 Skill

**用途**: 自动抓取飞书多维表格数据，生成前端流量分析HTML报告

**参数说明**:
- 无参数: 分析全部历史数据
- `--周度`: 分析最近7天
- `--月度`: 分析最近30天
- `--自定义 开始日期 结束日期`: 指定时间段（格式：YYYY-MM-DD）

---

### Step 2: 数据处理

- 按时间参数筛选数据
- 时间戳转换：毫秒 → 可读日期（如 1763568000000 → 2025/11/20）
- 计算汇总指标（求和、平均、最高、最低）
- 豆子÷10 = 金额（元）
- 识别异常值（退款率>平均2倍标红）

### Step 3: 生成HTML报告

**文件路径**: `reports/前端流量分析报告.html`（覆盖更新）
**备份路径**: `reports/history/前端流量分析报告_YYYY-MM-DD.html`

## 核心指标定义

### 概览页必须展示

**直播获客**:
- 直播场次 = 直播大盘数据记录数
- 总获客单量 = 直播大盘成交人数总和（非金额，因都是9.9）
- 场均获客 = 总获客/场次
- 分渠道明细：AI标准成交数、AI全域订单数、视频号成交人数

**ROI与消耗**:
- 抖音直接支付ROI = AI标准表「直接支付roi」平均值（关键指标）
- AI全域ROI = AI全域表「整体支付roi」平均值
- 视频号ROI = AI直播大盘「整体roi」平均值
- 小红书留资成本 = 总消费/总留资数

**消耗明细**:
- AI标准总消耗 = AI标准表「消耗」求和
- AI全域总消耗 = AI全域表「消耗」求和
- 视频号消耗 = AI直播投放「消耗总金额（豆）」求和 ÷10
- Dou+投放 = 视频投放dou+表「投放金额」求和

**其他关键指标**:
- 高消耗素材数 = R4+R5数量
- 主题直播场次 = 主题直播预约表不同主题数
- 最高预约人数
- 小红书总留资

## HTML Tab结构（7个Tab）

### Tab 1: 核心概览
- 直播获客卡片组
- 渠道分布可视化条
- ROI与消耗卡片组
- 消耗明细卡片组
- 其他关键指标卡片组

### Tab 2: 抖音直播
- 账号说明（大号AI/蓝V求职）
- 直播大盘数据表（成交人数、金额、千次观看、人均观看、退款率）
- AI标准投放数据表（消耗、成交数、直接支付ROI、GPM）
- AI全域投放数据表（消耗、订单数、整体ROI、订单成本）
- 转化漏斗（曝光→观看→点击→成交）
- 流量结构（直播推荐/关注推荐/千川付费占比）
- 异常值监控（退款率>平均2倍标红）
- 相关性发现

### Tab 3: 视频号
- 抖音vs视频号对比表
- AI直播投放测试详情表（含实际消耗豆和实际消耗元）
- 投放测试小结
- 主题直播预约分析

### Tab 4: 小红书
- 核心指标卡片（总消费、总留资、平均成本、点击率）
- 笔记效果排名（按留资成本排序）
- 高效内容特征洞察

### Tab 5: 素材投放
- 素材消耗等级分布表（R0-R5）
- 素材问题诊断
- 素材优化建议

### Tab 6: 短视频
- Dou+投放汇总
- 投放策略说明

### Tab 7: 优化建议
- 优先级最高（本周）
- 中期优化（本月）
- 长期规划（季度）

## HTML设计规范

```css
背景渐变: linear-gradient(135deg, #1a1a2e, #16213e, #0f3460)
卡片背景: rgba(255, 255, 255, 0.03)
卡片边框: rgba(255, 255, 255, 0.08)
强调色-蓝: #4facfe
强调色-紫: #a855f7
正向/成功: #10b981
警告: #f59e0b
危险/异常: #ef4444
```

**组件规范**:
1. Header: 标题+副标题+数据周期标签
2. Tab导航: 7个Tab按钮，点击切换
3. 核心指标卡片: 带breakdown分解和进度条
4. 渠道分布条: 可视化占比
5. 数据表格: 带表头、hover效果
6. 转化漏斗: 可视化漏斗
7. 排名列表: 金银铜徽章
8. 洞察框: 4种类型（普通/成功/警告/危险）
9. 建议框: 渐变背景+序号列表

---

## 数据验证规则

1. 场次数 = 直播大盘数据记录数
2. 成交人数必须逐条相加验证
3. 日期需从时间戳转换，格式为 MM/DD
4. 豆子转金额 = 豆子数 ÷ 10
5. ROI保留2位小数
6. 百分比保留1位小数

---

## 输出要求

1. 生成完整HTML文件到 `reports/前端流量分析报告.html`
2. 创建带日期的备份到 `reports/history/`
3. 告知用户报告已生成，提示刷新浏览器查看
