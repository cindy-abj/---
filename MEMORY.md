### 前端流量分析Skill - 已完成固化

**已创建文件：**
- `reports/前端流量分析报告.html` - 主报告文件（固定路径，方便浏览器书签）
- `reports/history/前端流量分析报告_YYYY-MM-DD.html` - 历史备份
- `skills/流量分析.skill.md` - 固定化分析规范（v1.1.0）

**Skill命令：**
```
/流量分析              # 分析全部历史数据
/流量分析 --周度        # 分析最近7天
/流量分析 --月度        # 分析最近30天
/流量分析 --自定义 开始日期 结束日期
```

**分析维度（7个Tab，已固定）：**
1. **核心概览** - 直播获客卡片组、渠道分布可视化条、ROI与消耗、消耗明细、其他关键指标
2. **抖音直播** - 账号说明、大盘数据表、AI标准投放、AI全域投放、转化漏斗、流量结构、异常监控、相关性发现
3. **视频号** - 抖音vs视频号对比、投放测试详情（含豆→元转换）、投放小结、主题直播预约
4. **小红书** - 核心指标卡片、笔记效果排名、高效内容洞察
5. **素材投放** - 消耗等级分布（R0-R5）、问题诊断、优化建议
6. **短视频** - Dou+投放汇总、投放策略说明
7. **优化建议** - 本周/本月/季度优先级建议

**核心指标定义（已固化）：**
- 直播场次 = 直播大盘数据记录数
- 总获客单量 = 直播大盘成交人数总和（非金额，因都是9.9）
- 抖音直接支付ROI = AI标准表「直接支付roi」平均值
- 视频号消耗 = 豆÷10 = 元
- 高消耗素材 = R4+R5数量

**数据验证规则：**
- 时间戳需转换（毫秒→日期，如1763568000000 = 2025/11/20）
- 成交人数逐条相加验证
- ROI保留2位小数，百分比保留1位小数

**修改流程：**
1. 如需调整分析维度或HTML格式 → 修改 `skills/流量分析.skill.md`
2. 重新执行 `/流量分析` 验证效果
3. 确认无误后Skill文件即为最新规范
